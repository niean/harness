# Skill: 回填-产品SENSE更新

触发：人工指令。经过一段时间的功能迭代后，人工触发本 Skill，让 Agent 从知识库和近期变更中自动提取产品判断准则的变化，经人工确认后更新 01-prd-sense.md（该文件位于 users/ 目录，需人工确认方可修改）。

## 步骤

### Step 1 — 读取现状

读取以下文件，了解产品定位和近期变化：
- .harness/context/users/01-prd-sense.md（当前产品定位与判断准则）
- .harness/context/users/01-prd-baseline.md（稳定需求基线）
- .harness/context/users/01-prd-specs.md（迭代演进记录，重点看最近几个版本）

### Step 2 — 提取候选变更

对比 prd-specs 近期迭代内容与 01-prd-sense.md 现有描述，识别可能需要更新的内容，例如：
- 新增了产品不做的方向（不做什么）
- 体验原则发生变化
- 判断准则的优先级或条目有调整

列出候选变更清单，每项注明：变更位置、当前描述、建议新描述、对应需求来源。

### Step 3 — 等待人工确认

通过 ask_followup_question 工具向用户展示候选清单（候选内容必须写在 question 参数中，不能写在工具调用之前的文本里，否则用户看不到）。等待用户确认哪些需要落盘，哪些不需要。

### Step 4 — 更新 01-prd-sense.md

按用户确认的候选项，最小化修改 01-prd-sense.md：
- 保持既有文风（普通文字，禁止 emoji/加粗/斜体）
- 只更新有实质变化的条目，不改变整体结构

### Step 5 — 输出变更摘要

向用户输出：修改了哪几处、对应哪个需求版本。
