# Skill: 提取-Skill

触发：人工指令。经过一段时间的 AI Coding 后，人工触发本 Skill，让 Agent 从近期编码活动中发现可复用的操作模式，抽象为新的 Skill。

## 步骤

### Step 1 — 分析近期活动

回顾近期 git 提交历史和编码活动（`git log --oneline -20` 等），识别重复出现的操作模式，例如：
- 多次执行的相似流程（如特定类型的重构、迁移、检查）
- 跨多次 Task 反复出现的步骤组合
- 已有 Skill 未覆盖但频繁发生的操作

### Step 2 — 提出候选 Skill

列出候选 Skill 清单，每个候选项包含：
- 名称
- 触发方式（人工指令 / 自动触发 / 混合）
- 解决什么问题、覆盖哪些重复操作
- 预估步骤概要

输出候选清单，等待用户确认哪些需要落盘。

### Step 3 — 读取参考格式

读取 `.harness/skills/` 目录下 1~2 个现有 Skill 文件，确认格式规范：
- 标题：`# Skill: <名称>`
- 触发说明
- 步骤列表（按执行顺序编号，每步目标明确、可执行）

### Step 4 — 生成 Skill 文件

按用户确认的候选项，在 `.harness/skills/` 目录下创建 `.md` 文件。内容包含：
- 标题和触发说明
- 分步骤描述
- 如涉及 subagent 调用，注明 prompt 模板路径
- 如涉及后续动作（构建验证、知识回填等），在最后一步注明

### Step 5 — 注册到 AGENTS.md

在 AGENTS.md 的 Skills 表格中新增对应行，包含：Skill 名称、触发方式、文件路径。

### Step 6 — 输出确认

向用户输出：新建文件路径、各 Skill 摘要、AGENTS.md 变更内容。
