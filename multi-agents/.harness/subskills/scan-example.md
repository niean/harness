## 检查规则

{{CHECK_RULES}}

示例（架构边界类）：
1. UI 层不直接调用底层服务 -- UI 层不直接引用底层 API，须通过 Coordinator 或 Manager。
2. 网络请求设超时 -- 必须设超时，不允许无超时请求。
3. 异步操作非主线程 -- 后台执行，回调切回主线程更新 UI。

示例（编码约定类）：
1. 常量集中管理 -- 常量写入常量文件，不散落各文件，不硬编码魔法值。
2. 文档禁止润色 -- 注释和文本禁用 emoji/加粗/斜体。

示例（安全规范类）：
1. 密钥只存安全存储 -- API Key 等只存 Keychain，不硬编码、不写日志。
2. 强制 HTTPS -- 外部 API 全 HTTPS，不降级。
3. API 响应校验 -- 校验状态码和数据完整性，解码失败按错误处理。

示例（日志规范类）：
1. 统一日志框架 -- 禁止 print()，统一使用日志框架。
2. 日志禁止输出敏感信息 -- 仅输出脱敏信息。

示例（废弃代码类）：
1. 未使用的类型/函数/变量 -- 已定义但未被引用。
2. 无效导入 -- import 的模块在当前文件未使用。
3. 过期注释 -- 注释引用了不存在的类型/函数/文件名。
